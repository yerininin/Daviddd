## 01. 마케팅 활용 케이스

기존 촬영 원본의 부분 편집 (릴리즈)과 A to Z 전체 생성 (개선 진행 중), 두 가지 트랙을 운영합니다.

### Case 1: Edit — Released

기존 촬영 원본을 AI로 부분 변경하여 활용도를 극대화하는 방식. 원본 촬영 이미지를 베이스로 하기 때문에 **품질 안정성이 높고**, 현업에서 실제 마케팅 소재로 릴리즈 중입니다.

| 워크플로 | 설명 | 활용 현황 |
| --- | --- | --- |
| **배경 변경** | 국내 촬영 컷의 배경을 브랜드 무드에 맞는 장소로 자유롭게 변환하여, 로케이션 섭외의 물리적 한계와 비용을 획기적으로 해결 → 인물 보존·조명·원근 불일치 시 자동 검수 탈락 → 프롬프트 강화 후 재생성 | **Released** |
| **착장 변경** | 현장 촬영 후에도 착장을 자유롭게 교체하여, 재촬영 없이도 전 상품 라인업의 화보를 즉각 확보 → 착장 디테일·얼굴 동일성 불일치 시 자동 검수 탈락 → 프롬프트 강화 후 재생성 | **Released** |

> **현업 릴리즈 중** — 원본 촬영 이미지를 베이스로 하기 때문에 품질 안정성이 높고, 현업에서 실제 마케팅 소재로 릴리즈 중

### Case 2: Total Generation — In Progress

A to Z 전체를 새로 생성하는 방식.

| 워크플로 | 설명 | 현업 평가 |

| **AI 인플루언서** | 표정·포즈·배경 프롬프트 라이브러리 조합으로 인플루언서 스타일 이미지 생성 → 자연스러움·포즈 정확도 자동 검수 → 탈락 시 프롬프트 강화 후 재생성 | |

## Solution: 프롬프트 라이브러리

생성 퀄리티 문제의 근본 원인은 **프롬프트 품질의 편차**입니다.
결과물 품질이 불균일해지는 문제를, **검증된 프롬프트 라이브러리**로 해결합니다.

## 02. 프롬프트 라이브러리

마케팅 본부에서 전달받은 레퍼런스 데이터를 **표정, 포즈, 배경** 3개 카테고리로 분류하여 프롬프트 라이브러리 DB로 데이터화 완료.

### 2.1 마케팅 본부 데이터의 프롬프트 라이브러리 데이터화

| 카테고리 | 수량 | 데이터화 내용 | 진행 현황 |
| --- | --- | --- | --- |
| **표정** | 10 | 감정/분위기별 표정 (시크 5 / 러블리 5) | 1차 완료 |
| **포즈** | 75+ | 전신/상반신/앉은포즈/거울셀카 (관절 단위 기술) | 1차 완료 |
| **배경** | 96+ | 9개 카테고리 (핫플카페/그래피티/철문/해외거리 등) | 1차 완료 |
| **촬영 기법** | - | 프레이밍, 렌즈, 앵글, 구도, 조리개 | 진행 예정 |
| **스타일링** | - | 코디 방법 (레이어링, 묶어 입기 등) | 진행 예정 |

> **181+ 프롬프트 라이브러리** 축적 완료

### 2.2 운영 방안

마케팅 본부가 레퍼런스 이미지를 수집·전달 → AX팀이 VLM으로 데이터를 추출하여 프롬프트 라이브러리에 축적 → 지속 보완하는 순환 구조.

- **1차 완료**: 표정 · 포즈 · 배경
- **예정**: 톤앤매너 · 촬영기법 · 구도 · 색감

#### F&F Studio Step 2: 5-Step Pipeline

사용자의 자연어 요청 → AI 생성 엔진 → 브랜드 자산 축적 지능형 파이프라인

| Step | 단계 | 내용 |
| --- | --- | --- |
| 01 | 사업부 입력 | F&F Studio (Claude Code), User Request, Reference Image |
| 02 | 탐색 | MCP: 유저의 요청에 따라 워크플로우 조합 |
| 03 | 생성 | Image Gen Skill, Claude Code, Content Library |
| 04 | 피드백 | 피드백 갤러리: 결과 이미지 확인, 피드백 전달 |
| 05 | 자산화 | 컨텐츠 라이브러리: 브랜드별 데이터화해서 저장, 자동으로 생성 Skill 개선 |

#### 프론트엔드 인터페이스

마케팅 본부가 직접 사용하는 요청서 작성 화면과 결과물 피드백 화면.

| 화면 | 설명 |
| --- | --- |
| **요청서 작성** | 브랜드 선택 → 무드/착장/얼굴 이미지 업로드 → 자연어로 지시사항 입력 |
| **피드백 갤러리** | Release → 즉시 다운로드 |

- **마케팅 본부**: 생성된 이미지를 갤러리에서 확인, Release 클릭 → 고해상도 즉시 다운로드, 추가 의견·태그로 선호도 피드백
- **AX팀**: Release/Reject 데이터 분석, 선호 패턴 학습 → 프롬프트 라이브러리 보정, 탈락 사유 기반 검수 기준 강화, 다음 생성 품질에 자동 반영

## 03. Core Architecture — 클로드코드

Claude Code Skill 안에 코어 AI 파이프라인을 구축 — 앞서 구축한 프롬프트 라이브러리와 각 워크플로를 Claude Skills로 연동하여, 레퍼런스 이미지 분석부터 이미지 생성, AI 검수, 재시도까지 자동으로 수행합니다.

### 파이프라인

| Step | 단계 | 내용 | 담당 |
| --- | --- | --- | --- |
| 01 | 인풋 | 워크플로 지시 (배경 변경, 착장 변경, 브랜드컷, AI 인플루언서) | 사용자 |
| 02 | AI 프롬프트 설계 | 인풋 분석 → 라이브러리 조합 | AI (자동) |
| 03 | 이미지 생성 | 완성 프롬프트 + 인풋 이미지 → Gemini 이미지 생성 모델 | AI (자동) |
| 04 | 자동 검수 → 재생성 루프 | 일관성 유지 → 포즈, 표정, 배경, 촬영기법, 코디방식 → 브랜드톤 품질 검수. 탈락 시 원인 진단 → 프롬프트 수정 → 재생성 (최대 2회) | AI (자동) |

### 워크플로는 프롬프트 모듈 설계 조합으로 만들어짐

VLM 분석기 + 프리셋 라이브러리 + 브랜드 규칙 → 프롬프트 빌더가 조합하여 모든 워크플로의 프롬프트를 생성

```
VLM 분석기 (포즈, 착장, 표정, 배경, 얼굴, 헤어+메이크업, 컨셉/무드, 조명/색)
         ↓
   프롬프트 빌더  →  워크플로 (배경변경, 착장변경, 브랜드컷, AI 인플루언서, 개발예정...)
         ↑
프롬프트 프리셋 (포즈, 배경, 스타일, 시나리오)
브랜드 규칙 (MLB DNA, 금지조합, 네거티브)
```

> 이 모든 기능이 Claude Code Skill 안에 내장 → 새로운 워크플로를 빠르게 조합해 생성 가능

## 04. Future Strategy — 향후 전략

이커머스 채널 자동화와 지식그래프 연계를 통한 AI 기획까지 확장합니다.

### 4.1 이커머스 확장 (Figma MCP 연동)

Claude Code에 **Figma MCP**를 연동하여, 브랜드별 상세페이지 규칙과 채널별 배너 규칙을 **스킬(Skill)**로 문서화하고 AI 이미지 생성부터 채널 배너 일괄 생성까지 **E2E 자동화 파이프라인**을 구축합니다.

| Step | 내용 |
| --- | --- |
| Step 1 | 상세페이지 자동 생성: 브랜드별 상세페이지 규칙을 스킬 문서화하여 자동 세팅 |
| Step 2 | 채널별 배너 자동 생성: 채널별 배너 사이즈/레이아웃 규칙을 스킬 문서화하여 자동 생성 |
| Step 3 | E2E 파이프라인: AI 이미지 생성 → 상세페이지 세팅 → 채널 배너 일괄 생성 |

### 4.2 지식그래프 연계

매출·검색량·트렌드 데이터를 연계한 **지식그래프**로 컨텐츠 기획부터 소재 자동 제작까지 확장합니다.

| 단계 | 내용 |
| --- | --- |
| 지식그래프 (Data) | 매출 분석, 검색량 추이, 트렌드, 소셜 분석 |
| 자동 기획 (Plan) | 컨텐츠 방향, 소재 선정, 타겟 제안 |
| AI 생성 (Execute) | 프롬프트 라이브러리 자동 선택/조합, 생성 + 검수, 채널별 소재 자동 제작 |

**목표**: "어떤 상품을, 어떤 컨셉으로, 어떤 채널에" → 데이터가 자동 제안하고, AI가 생성하고 채널별 소재 자동 제작
